# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm start        # production — node server.js
npm run dev      # development — nodemon (auto-restart on changes)
```

No build step, no tests, no linter.

## Architecture

Single-file Node.js server + vanilla HTML/CSS/JS frontend. No framework, no bundler.

### Server (`server.js`)

All backend logic lives here. Rooms are stored in a `Map` in process memory — **no persistence**. A room is deleted 1 hour after its last participant leaves.

Key design points:
- Cards are **personalized per viewer**: `viewCard()` strips the author name unless `room.revealed` is true or the card belongs to the viewer. `broadcastCard()` iterates every participant and sends each their own view via `io.to(pid)`.
- The first socket to create a room becomes the **facilitator**. On disconnect, facilitator role transfers to the next participant in insertion order.
- Room codes are 6-char alphanumeric (ambiguous chars `0`, `O`, `1`, `I` excluded), generated by `generateCode()`.

Retro formats are declared in the `FORMATS` constant at the top of `server.js`. Adding a new format only requires adding an entry there — the rest of the system is data-driven.

### Frontend

| File | Role |
|---|---|
| `public/index.html` | Home page — create/join forms, Socket.io logic inlined in a `<script>` tag |
| `public/room.html` | Room page shell — only markup; all logic is in `room.js` |
| `public/room.js` | Room client — local state mirror, rendering, socket event handlers |
| `public/style.css` | All styles |

`room.js` maintains a local `state` object (`state.cards`, `state.participants`, etc.) that mirrors the server. Socket events update this state and patch the DOM directly — no virtual DOM or reactivity layer.

**Room entry flow**: the index page emits `create-room` or `join-room`, stores the returned room snapshot in `sessionStorage`, then redirects to `room.html`. `room.js` reads `sessionStorage` on boot; if absent (direct link), it shows the name modal and calls `join-room` itself.

**Keyboard shortcuts in forms**: `Ctrl/Cmd+Enter` submits; `Escape` cancels. Applies to both the add-card form and the inline card editor.
